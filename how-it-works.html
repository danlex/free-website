The flow

From clone to live site in six steps.

1

Clone the repo

Start with the repo as-is. If you want the default path, use the root files. If you want a different stack, run the generator script. 

git clone https://github.com/your-username/your-repo-name.git
cd your-repo-name
2

Pick the simplest starter that fits

Most people should start with no framework. If needed, run the wizard and generate an Astro or Next.js version. 

chmod +x scripts/create-site.sh
./scripts/create-site.sh
3

Open it in Codex and customize

Rewrite the copy, change the colors, add your links, and make the site feel specific to the person using it.

Good prompt: “Turn this into a personal website for [name], a [role]. Keep it compatible with GitHub Pages.”

4

Push to GitHub

Create a new repo, commit the files, and push `main`. Framework-based starters already include a Pages workflow. 

git init
git add .
git commit -m "Initial website starter"
git branch -M main
git remote add origin https://github.com/your-username/your-repo-name.git
git push -u origin main
5

Enable GitHub Pages

No-framework sites deploy from the root branch. Astro and Next.js templates deploy via GitHub Actions.

  • Open `Settings` → `Pages`
  • Choose `Deploy from a branch` for no framework
  • Choose `GitHub Actions` for Astro or Next.js
6

Connect the custom domain

Rename `CNAME.example` to `CNAME`, put the domain in the file, update DNS, then enable HTTPS in GitHub Pages.

Domain setup

The last 10 percent that makes it feel real.

Subdomain

For `www.yourdomain.com`, add a `CNAME` record pointing `www` to `your-username.github.io`.

Apex domain

For `yourdomain.com`, add GitHub Pages `A` records and, if you want, `AAAA` records for IPv6.

GitHub setup

Add the custom domain in Pages settings and enable HTTPS once DNS has propagated.
