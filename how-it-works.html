Step by step

Do these 5 things in order.

1

Create your own repo on GitHub

Open this repo on GitHub and click Use this template. If template mode is not enabled, fork the repo instead.

Result: you now have your-username/your-repo-name on GitHub.

Screenshot 1 GitHub repo page showing Use this template or Fork.
2

Clone your repo

Clone your own repo, not this public repo. Work from your own repository from this step onward. 

git clone https://github.com/your-username/your-repo-name.git
cd your-repo-name

Result: the project is now on your machine.

Screenshot 2 Terminal showing the clone command and the project folder open.
3

Open it in Codex or Claude Code

Open the repo in your coding agent. The main files you will change are: 

index.html
styles.css
script.js

Paste this prompt: “Turn this starter into a landing page for [business or product]. Create a strong hero, benefits, proof, CTA, FAQ, and contact section. Keep it plain HTML, CSS, and JavaScript.”

Using Codex, you can also generate images directly. Example: “Generate a clean hero image for a landing page about [business or product]. Style: modern, minimal, premium, white background with warm orange accents. Format: wide website hero image.”

Result: the agent rewrites the starter into your landing page.

Screenshot 3 Codex or Claude Code opened in the repo with the page prompt visible. Optional: show the image generation prompt too.
4

Push your changes to GitHub

After you review the edits, commit them and push them to your own repository. 

git add .
git commit -m "Customize landing page"
git push

Result: your updated files are now on GitHub.

Screenshot 4 GitHub showing the updated files or latest commit on main.
5

Enable GitHub Pages

In your GitHub repo, open the Pages settings and publish from the root of the main branch. 

Settings > Pages
Deploy from a branch
Branch: main
/ (root)

Result: your landing page gets a live GitHub Pages URL.

Screenshot 5 GitHub Pages settings showing main and / (root).

Optional domain

Add your domain after the site is live.

1. Add the domain in GitHub

In Settings > Pages, add your custom domain. GitHub can create the CNAME file for branch publishing, or you can rename CNAME.example to CNAME yourself.

2. Update DNS

For www.yourdomain.com, create a CNAME to your-username.github.io. For an apex domain like yourdomain.com, use GitHub Pages A records.

3. Wait for HTTPS

DNS can take time to propagate. When GitHub shows the option, enable HTTPS.

Important: GitHub Pages hosting can be free for public repositories. Your custom domain is usually a separate paid purchase from your domain provider.

Screenshot 6 GitHub Pages custom domain field and your DNS provider records.
