Competitor Snapshot

How this starter compares with popular website builders.

Reviewed on April 11, 2026 from official docs and pricing pages.

This starter

Codex + GitHub Pages

  • Static HTML/CSS/JS and full code ownership
  • Branch publishing from main and / (root)
  • Custom domain and HTTPS supported on GitHub Pages
  • No mandatory platform subscription for hosting

Carrd

Fast one-page builder

  • Custom domain publishing requires Pro Standard or higher
  • Free/entry path is optimized for Carrd-hosted URLs
  • Great for speed, lower raw code control

Framer

Design-first editor

  • Free plan exists, but custom domain requires paid plan
  • Strong visual workflow and built-in CMS options
  • Good fit for design-heavy marketing teams

Webflow

Visual CMS platform

  • Starter plan publishes to webflow.io
  • Custom domain launch requires a paid Site plan
  • Powerful CMS and collaboration, steeper complexity

Sources: GitHub Pages publish source, GitHub custom domain, Carrd custom domain requirements, Framer pricing, Webflow pricing.

Choose this starter when

  • You want full source ownership and portability
  • You want free hosting and your own domain setup
  • You prefer prompting Codex over visual dragging

Choose a builder when

  • You need visual editing by non-technical teammates
  • You need built-in CMS and editor permissions immediately
  • You accept platform plan costs as the speed tradeoff

FAQ

Short answers for the obvious questions.

Is this really static?

Yes. This repo uses plain HTML, CSS, and vanilla JavaScript only.

Do I need to keep this exact layout?

No. This page is the starter. Replace sections, remove what you do not need, and rearrange it however you want.

Do I need a build step?

No. The default flow is branch-based GitHub Pages publishing with no build tooling.

Can I preview the site before I publish?

Yes. Run sh scripts/preview.sh and open http://localhost:8000.

Is the custom domain optional?

The product promise includes a custom domain. GitHub Pages hosting can stay free, but the domain itself is usually a separate purchase.

What is actually free here?

The starter, GitHub repo, and GitHub Pages hosting can stay free. What usually costs money is the domain you want to connect.

Should I verify my domain in GitHub first?

Yes. GitHub recommends verifying the custom domain before you add it to the repository so you reduce takeover risk and keep ownership clearer.

What should I ask Codex to do first?

Start with prompts/landing-page.md unless you already know you want a portfolio or personal website structure.
